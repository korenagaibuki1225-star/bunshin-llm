# 分身LLM — 自分の全会話データで「自分の分身」を作る

自分のLINE・AIチャット履歴・手書き日記から学習データを構築し、小型LLMをファインチューニングして
「自分の口調で、相手との関係に応じて話し方を切り替える分身AI」を作るプロジェクト。

データ収集 → 前処理 → プライバシー処理 → 学習 → 評価 → 改善のMLライフサイクルを、
**生成AI（ClaudeCode）と一緒に・無料の計算資源だけで**一周した記録です。

## デモ(分身の実際の返答)

```
[相手(友達)] 今日も学校疲れた〜
[分身] あ
       あと、一応、明日は12:00ぐらいから学校に行くよ!
```
LINE特有の「短文を分けて連投する」クセや、相手が母のときだけ絵文字が出る、
といった本人の癖を再現している(データは非公開のため、出力例のみ掲載)。

## アーキテクチャ

```
LINE / ChatGPT / Claude / Gemini / 手書き日記(写真→文字起こし)
        │
        ▼ make_dataset.py
  形式自動判別 → クリーニング → 60分ルールで会話分割 → 関係ラベル付け
  → 直前6ターンの文脈つき学習例(チャット形式JSONL)→ ソース比率調整
        │
        ▼ anonymize.py
  実名→仮名の一括置換(700箇所超)+ 残存ゼロの自動検証
  ※クラウドに上がるのは仮名版のみ
        │
        ▼ bunshin_lora.ipynb(Kaggle T4 / 無料)
  Qwen3.5 + QLoRA(4bit凍結 + LoRA r=16)
  systemプロンプトで「相手との関係」を条件付け
  train_on_responses_only(自分の返事だけを学習)
        │
        ▼
  評価(固定10問 + ブラインドテスト)→ 診断 → 次バージョンへ
```

## 改善の記録(このプロジェクトの本体)

| 版 | 構成 | 結果 | 診断と対策 |
|---|---|---|---|
| 1号 | 4B・単発ペア2,955組 | 口調は再現、返事が支離滅裂 | 文脈ゼロのデータが原因 → マルチターン化 |
| 2号 | 4B・文脈つき | 関係による口調切替に成功。短文挨拶に弱く、検証損失もまだ下降中 | テスト条件のミスマッチ + 学習不足 → 連続会話評価・9B化・5epoch |
| 3号 | 9B・5epoch | (学習中) | |

副産物として、事前学習の基礎を体で理解するために**文字レベルGPT(1.66M)をゼロから実装**し、
同じ損失カーブの読み方で「過学習」(検証損失の反転)と「学習不足」(下降中の打ち切り)の
両方を自分のデータで観測した(`mini_gpt/`)。

## プライバシー設計

- 生データ・生成データセット・学習済みモデルは**一切公開しない**(.gitignoreで物理的にブロック)
- クラウド(Kaggle)にアップロードするのは実名を仮名化したデータのみ
- 仮名化は置換後に「元の名前が1件でも残っていないか」を自動検証
- 学習済みモデルは本人の私的データを記憶しているため、ローカル推論のみで運用

## リポジトリ構成

```

  colab/           学習ノートブック(Colab版 / Kaggle版)
evaluation/        評価プロトコル(固定10問 + ブラインドテスト)
WORKFLOW.md        全工程の詳細な記録
```

## 自分で再現するには

1. 各サービスから自分のデータをエクスポートし `bunshin_data/raw/` に配置(README参照)
2. `python make_dataset.py` → `python anonymize.py`
3. `name_map.example.json` を参考に自分用の `name_map.json` を作成
4. `colab/bunshin_lora_kaggle.ipynb` をKaggleで実行
