# 分身LLMができるまで — 初学者による、初学者のための全記録

このドキュメントは、プログラミング学習中の大学生(私)が、AIアシスタント(Claude)と
ペアを組んで「自分の分身LLM」を作った全工程の記録です。
専門用語には必ず「やさしい説明」を付けています。私自身が初学者だからです。

---

## フェーズ0: そもそもLLMって何? — 自分で作って理解する

**やったこと**: 約150行のPythonコードで、ミニチュアのGPT(パラメータ166万個)をゼロから作り、
青空文庫の小説55万文字で学習させた(`mini_gpt/`)。

**やさしい説明**: LLM(大規模言語モデル)の正体は「**次の1文字を予測するマシン**」です。
「吾輩は猫で」まで読んだら次は「あ」だろう、と当て続けるだけ。これを何百万回も練習すると、
文章が書けるようになります。ChatGPTも原理はこれと同じで、規模が約10万倍なだけ。

**学んだ専門用語**:
- **トークナイザ** = 文字を数字に変換する辞書(AIは数字しか読めない)
- **損失(loss)** = 予測の「間違い度」。学習が進むとこの数字が下がる
- **過学習(overfitting)** = 練習問題を丸暗記してしまい、初見の問題が解けなくなること

**一番の発見**: 学習データを小説から「自分の文章12万字」に変えたら、意味不明ながらも
「〜やけな」「対策を教えて」など、**私の口癖の面影がある文章**を吐くようになった。
データを変えれば人格の匂いが変わる — 分身づくりの原理を最小サイズで確認できた。

## フェーズ1: 材料集め — 自分の「言葉」を全部かき集める

**やったこと**: 過去3年ぶんの自分の言葉を5つのルートから回収した。

| どこから | どうやって | 何が取れた |
|---|---|---|
| LINE(10人分) | 公式のトーク履歴送信機能 | 実際の会話690往復 |
| ChatGPT | 設定→データエクスポート | 1,820往復 |
| Claude | Settings→Export data | 445往復 |
| Gemini | Google Takeout | 質問文1,065件 |
| 手書きの日記30ページ | スマホで撮影→AIが文字起こし | 約2.9万字の内面 |

**やさしい説明**: 分身の「らしさ」はプログラムではなくデータから生まれます。
AIチャットの履歴には「考え方」、LINEには「友達との素の口調」、日記には「内面の声」が
入っていて、この3種類が揃うと人格に立体感が出るとおもいます！(個人の意見)

**ハマったポイント**(重要な学び):
- サービスごとにデータの形式がバラバラ。ChatGPTは木構造、Geminiは操作ログ形式
- Geminiの会話はTakeoutの「Gemini」ではなく「マイ アクティビティ」に入っている(罠!)

## フェーズ2: 下ごしらえ — 会話を「教科書」に変える

**やったこと**: `make_dataset.py` を作り、集めた生データをAIが学習できる形式(JSONL)に自動変換。

**やさしい説明**: AIの教科書は「相手がこう言った → 私はこう返した」というペアの集まりです。
ただし生データをそのまま使うとゴミだらけになるので、下ごしらえをします:

1. スタンプ・通話記録・URLだけの行を捨てる(ノイズ除去)
2. 1時間以上空いたら別の会話として区切る(昨夜の「おやすみ」と今朝の「おはよう」を混ぜない)
3. 各ペアに「相手は友達/母/先生」のラベルを付ける → **相手によって話し方を変えることまで学習させる**
4. 返事の前に「直前6ターンの会話の流れ」を付ける(1往復だけだと文脈無視の分身になる — 1号の失敗から学んだ)
5. AI由来のデータが77%と偏っていたので800件に間引く(敬語ばかりの分身になるのを防ぐ)

**学んだ専門用語**: この工程は業界で「**データキュレーション**」と呼ばれ、
機械学習の仕事の8割はここだと言われています。実感として、本当にそうでした。

## フェーズ3: プライバシーの壁 — 実名を仮名に

**やったこと**: `anonymize.py` で、データ中の実名743カ所を自然な仮名に自動置換。
置換後に「元の名前が1件でも残っていないか」を機械的に検証してから、
仮名版だけをクラウド(Kaggle)にアップロードするルールにした。

**やさしい説明**: 学習データには家族や友達の名前が入っています。学習済みのAIは
データを「記憶」してしまうことがあるので、(1)クラウドに上げる前に仮名化、
(2)完成した分身モデル自体も非公開、という二重の防御にしました。
このリポジトリにコードだけがあってデータとモデルが無いのは、手抜きではなく設計です。

## フェーズ4: 学習 — 巨人の肩に乗る

**やったこと**: ゼロから作るのではなく、日本語をすでに話せる公開モデル
(Qwen3.5、40億→90億パラメータ)を借りてきて、「私らしさ」だけを追加学習した。

**やさしい説明**:
- **ファインチューニング** = 転校生(できあいの優秀なモデル)に、うちのクラスの空気(私のデータ)だけを教えること。ゼロから育てるより圧倒的に効率的
- **LoRA** = モデル全体(90億個のダイヤル)は凍結して、小さな「差分メモ」だけを書き換える省エネ学習法。無料GPUでも動く理由
- **QLoRA** = さらにモデルを4bit(超圧縮)で持つ工夫。これで16GBのGPUに90億パラメータが載る

**インフラとの戦い**(全部本当にあったこと):
- Colab無料枠 → ドライブ接続失敗の連発 + GPU使用上限 → **Kaggleに移住**(週30時間無料)
- 手作業に疲れて → **Kaggle APIで学習ジョブを自動投入**する方式に進化
- 自動実行の初回、旧世代GPU(P100)が割り当てられて即死 → メタデータに `machine_shape: NvidiaTeslaT4` を指定して解決
- 9Bモデルの変換で作業ディスク20GBを突破して死 → 変換作業を広い/tmpに逃がして解決

## フェーズ5: 評価と改善 — このプロジェクトの本体

**3世代の記録**:

| 版 | 構成 | 結果 | 診断 |
|---|---|---|---|
| 1号 | 4B・単発ペア | 口調は再現、返事が支離滅裂 | データに文脈が無かった |
| 2号 | 4B・文脈つき・3周 | 関係による口調切替に成功。短文に弱い | 検証損失がまだ下降中=**学習不足** |
| 3号 | 9B・文脈つき・5周 | 「明日ひま?」→「ちょいまち」— 会話が成立し始めた | 3周目が頂点、以降は**過学習** |

**3号の検証損失カーブ**(数字は「初見データでの間違い度」):
```
1周目: 2.57 → 2.44 → 2.33   ぐんぐん賢くなる
3周目: 2.13 → 2.12 ★頂点
4周目: 2.39 → 2.33 → 2.34   丸暗記が始まる
5周目: 2.51 → 2.52           初見に弱くなっていく
```

**やさしい説明**: 2号では「勉強が足りない」を、3号では「勉強しすぎて丸暗記」を、
同じグラフの読み方で観測しました。ちょうどいい学習量(このデータなら3周)は
理論ではなくグラフが教えてくれます。頂点の瞬間のモデル(checkpoint)が保存してあったので、
5周やっても3周時点のベスト状態を取り出せました。

**評価の仕組み**: 感想戦にならないよう、固定10問(`evaluation/questions.md`)を毎世代に
同じ条件で出題して比較します。仕上げは家族による「どっちが本人?」ブラインドテストです。

## この記録を読んでいる初学者のあなたへ

このプロジェクトは「cdコマンドって何?」というレベルから始まりました。
必要だったのはAIの前提知識ではなく、(1)自分のデータという材料、(2)エラーをそのまま貼って
