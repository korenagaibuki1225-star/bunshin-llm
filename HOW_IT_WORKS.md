# 分身LLMのしくみ — 初学者のための解説

「この分身、中では何が起きてるの?」への答え。
データ分析の教科書に出てくる言葉(ワンホットエンコーディング、主成分分析、softmax…)と
つなげて説明します。私自身がその教科書で勉強中だからです。

---

## 全体像:分身は「7段の変換装置」

```
①文字→番号 → ②番号→意味の座標 → ③文脈を混ぜる(×何十層)
→ ④次の言葉の確率 → ⑤サイコロで1語選ぶ → ①に戻って繰り返し
```

返事が丸ごと出てくるのではなく、**「次の1トークン(言葉のかけら)」を選ぶ作業の繰り返し**で
文章が生まれます。これが生成AIの正体です。

---

## ① トークン化:文字を番号にする

「おつー」→ `[3204, 891]` のような番号列に変換します。AIは数字しか扱えないからです。
変換表(トークナイザ)は辞書のようなもの。

## ② 埋め込み:実はワンホットエンコーディングが隠れている

番号のままだと「891番と892番は意味が近い」という誤解が生まれるので、
各トークンを**数千個の数字の組(ベクトル)**に変換します。

ここで教科書の知識が繋がります。この変換は数学的には

> **ワンホットベクトル × 巨大な行列 = 埋め込みベクトル**

とまったく同じ。つまりワンホットエンコーディングは**第1層に埋まっています**。
違いはその先で、ワンホットの「全単語が互いに無関係」という表現を、
**意味が近い言葉同士が近くに並ぶ「意味の地図」**に写し変えます。
「嬉しい」と「楽しい」がご近所さんになる、数千次元の地図です。

### コラム:主成分分析(PCA)との関係

PCAは「たくさんの変数を、情報を保ったまま少数の軸に圧縮する」手法。
埋め込みがやっているのはその親戚です — 数万語彙を数千次元の「意味の軸」へ圧縮している。

| | PCA | 埋め込み(深層学習) |
|---|---|---|
| 変換 | 線形のみ | 非線形もOK |
| 軸の決め方 | 分散最大の方向を一発計算 | 「次の言葉の予測が当たるように」逆伝播で徐々に学習 |

**「目的に合わせて自動で作られる、非線形版PCAのような表現」**と覚えると、
統計の知識とすっきり繋がります。

## ③ 自己注意機構:文脈を織り込む(Transformerの心臓)

各単語が3つの顔を持ちます:

- **Q(クエリ)**「私はこういう情報を探しています」
- **K(キー)**「私はこういう情報を持っています」
- **V(バリュー)**「実際に渡す中身はこれです」

QとKの相性が良いペアほど強く「注目」され、そのVを多く受け取る。
注目度は合計1の確率として計算されます。
「夏休みは**帰って**くる?」の「帰る」が「実家」の意味を帯びるのは、
この仕組みが文中の言葉同士を結びつけるからです。
これを何十層も重ねて、文脈が深く染み込んだ表現にしていきます。
(このリポジトリの `mini_gpt/step3_gpt.py` に、この機構の最小実装があります)

## ④ softmax:スコアを確率に変える

最後の層が「次のトークン候補、全語彙ぶんのスコア」を出し、
**softmax**という関数で「合計1の確率分布」に変換します。
ロジスティック回帰の多クラス版、と言えば教科書と繋がるはず。

## ⑤ サンプリング:サイコロを振る

確率最大の1語だけを選ぶと毎回同じ優等生回答になるので、確率に従って
サイコロを振ります(`temperature=0.7` は冒険度、`top_p=0.9` は変な候補の足切り)。
分身の返事が毎回違うのはこのためです。

## ⑥ 学習:どこが「私」になったのか

- **事前学習(借り物)**: 土台モデル(Qwen3.5)は、大量のテキストで
  「次のトークン予測」を訓練済み。日本語力・一般常識はここから
- **LoRA微調整(私の追加分)**: 私の会話3年分で追加訓練し、
  「この文脈なら、私はこのトークンを選びがち」という方向に確率分布を歪ませた
- **私らしさの正体 = この確率の歪み**。口癖、方言、絵文字の癖、関係による使い分け、
  ぜんぶ「特定トークンの確率が上がる」という形で刻まれている
- 学習の物差しは**交差エントロピー**(予測分布と正解のズレ)。
  これを**逆伝播**で最小化する — 統計学の言葉がそのまま現場の言葉です

## ⑦ なぜ「違和感」があるのか(作者本人の感想より)

完成した分身と話した私の率直な感想は「違和感がすごい」でした。
仕組みを知るとその理由が説明できます:

- 分身が学んだのは**言葉選びの統計的なクセ**であって、記憶・出来事・意図ではない
- 「帰るわ!」のような定型のやりとりは統計で再現できるが、
  一貫した「自分」は統計の中に写っていない
- だから、表面は本人っぽいのに、芯が空洞 — この落差が違和感の正体

逆に言えば「口調が完璧でも本人と感じられないなら、**本人らしさはどこに宿るのか?**」
という問いが残ります。これは心理学の問いでもあり、
このプロジェクトの次の楽しみ(家族によるブラインドテスト)に続きます。

## 用語対応表:データ分析の教科書 ↔ 生成AI

| 教科書の言葉 | 生成AIでの居場所 |
|---|---|
| ワンホットエンコーディング | 埋め込み層の入口(ワンホット×行列=埋め込み) |
| 主成分分析(次元削減) | 埋め込み=学習で作る非線形版の低次元表現 |
| ロジスティック回帰・softmax | 出力層(次トークンの確率化) |
| 交差エントロピー | 損失関数(学習の物差し) |
| 勾配・最適化 | 逆伝播+AdamW(ダイヤルの回し方) |
| 過学習・汎化 | 検証損失カーブの読み方(WORKFLOW.md参照) |

データ分析の基礎は、生成AIの土台そのものでした。
