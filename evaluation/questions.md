# 分身の評価プロトコル

各バージョン(1号/2号/3号…)を同じ条件で比較するための固定評価セット。

## 評価1: 固定10問(バージョン間比較用)

連続会話モード(`chat()`)で、以下を毎回同じ順序で入力する。
relationは各問の指定に従う。生成のたびに結果は揺れるので、各問2回生成して良い方を記録。

| 3 | 友達(高校からの友達) | 最近なんかハマってることある? |

記録項目: 返答テキスト / 口調の本人らしさ(1-5) / 内容の噛み合い(1-5) / 破綻の有無

## 評価2: ブラインドテスト(本人らしさの人間評価)

1. 上記の質問から5問を選び、「本人の実際の返事」と「分身の返事」を用意する
2. 妹・友達など本人をよく知る審査員3名に、どちらが本人か当ててもらう
3. 正答率50%に近いほど分身の完成度が高い(見分けがつかない=成功)

## 3号の回答抜粋(固定10問より)

**うまくいった例**
- Q2「明日ひま?昼メシ行かん?」→「いいで!」「10:00ぐらいに集合しようかな」
- Q4「今日の約束リスケしてもいい?」→「ええで!むしろ歓迎や」
- Q3「最近ハマってることある?」→「散歩を毎日習慣にしてみたよ!」(本人の実際の習慣がデータからにじみ出た)

**課題が見えた例**
- Q7(ゼミの先生)「進捗はいかがですか?」→「ありがとうオリゴ糖😭」(敬語データが3件しかない弱点がそのまま出た)
- Q5(母)「夜ご飯何がいい?」→「こちらこそありがとう!」(母との会話データに感謝が多く、内容を無視して感謝で返すバイアス)
- 一部の回答に user / assistant などのロールタグが混入(生成の停止条件の調整が次の技術課題)


## 3号との自由会話・抜粋(インタラクティブ検証)

HFトークン導入で遊び場が安定稼働し、3つの相手モードで自由会話を実施。

**名場面(モード切替と本人性が機能した例)**
- 妹モード「夏休みは帰ってくる?」→「帰るわ!」
- 友達モード「夏休みは帰ってくる?」→「こっちは福岡やから無理そうや😔」(生活の地理感覚がデータから乗り移った)
- 先生モード「元気になる方法を教えてください」→「朝散歩を始めてみるのはどうでしょう」(敬語+本人の実際の習慣を提案)
- 妹モード「どんな仕事が向いてると思いますか?」→「まあ、努力は必要やな それぐらいや!」(兄の距離感)

**珍場面(限界の記録)**
- 「ありがとうオリゴ糖😭」が相手を問わず出現(定型フレーズの丸暗記。敬語データ不足も一因)
- 脈絡が切れると趣味の断片が漏れる(「ウマ娘の名前」等)
- 作者の感想:「違和感がすごい」— 口調の統計は写せても、一貫した自己までは写らない(詳細は HOW_IT_WORKS.md ⑦)
