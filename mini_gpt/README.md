# mini_gpt — 自分の分身LLMを作るプロジェクト

ゼロから小さいGPTを自作して、最終的に「自分の分身AI」を作るプロジェクト。

## 全体ロードマップ

| フェーズ | 内容 | 場所 |
|---|---|---|
| 1(今ここ) | 豆GPTをゼロから自作して仕組みを理解 | このPC(CPU) |
| 2 | 自分のデータ集め(LINE履歴・日記 → 会話ペアに整形) | このPC |
| 3 | 日本語対応の小型オープンモデルをLoRAファインチューニング | Google Colab(無料GPU) |
| 4 | できた分身をOllamaでこのPCで動かして会話 | このPC |

## フェーズ1の進め方

順番に実行する。各ファイルのコメントが教科書代わり。

```
cd mini_gpt
python data/prepare_data.py   # ステップ0: 学習テキストの準備(青空文庫、実行済み)
python step1_tokenizer.py     # ステップ1: 文字→数字の変換(トークナイザ)
python step2_bigram.py        # ステップ2: 最弱の言語モデル+学習ループ(1〜2分)
python step3_gpt.py           # ステップ3: 本物のGPT(自己注意機構)。CPUで20〜40分
python step3_gpt.py generate  # 学習済みモデルで生成だけやり直す(数秒)
```

## 各ステップで学ぶこと

- **ステップ1 トークナイザ**: LLMは文字を数字(トークンID)にしてから扱う
- **ステップ2 バイグラム**: 「次の文字を予測する」というLLMの本質と、学習ループ(予測→損失→逆伝播→修正)
- **ステップ3 GPT**: 自己注意機構(Q・K・V)、位置埋め込み、残差接続 — ChatGPTと同じ構造の縮小版(約1.6Mパラメータ)

## 自分のテキストで学習する(分身への第一歩)

`data/input.txt` を自分の文章(UTF-8のテキストファイル)で上書きして
`step3_gpt.py` を再実行するだけ。日記・ブログ・LINEのエクスポートなど、
量が多いほど良い(最低でも数十万文字が目安)。

## メモ

- このPCにはNVIDIA GPUがないため、学習はCPUで実行(このサイズなら問題なし)
- 本格的な分身(フェーズ3)はGoogle Colabの無料GPUを使う
- 参考: このコードは [karpathy/nanoGPT](https://github.com/karpathy/nanoGPT) の考え方をベースにした学習用ミニ版
